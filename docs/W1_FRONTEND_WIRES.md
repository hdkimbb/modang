# Week 1 — Frontend Pages & Wireframes

> 모당 (Modang) 프론트엔드 화면 설계

---

## 디자인 원칙

1. **모바일 퍼스트** — 당근이 모바일 중심이므로
2. **SEED 디자인 시스템 우선** — 자체 컴포넌트 최소화
3. **당근스러운 톤** — 따뜻한 주황, 둥근 모서리, 친근한 카피
4. **데이터 시각화 단순** — 차트보다 카드·리스트 위주

---

## 페이지 구조

```
모당 (Modang) 전체 페이지 트리

/
├── /                      # 홈 (Top Navigation + 모임 피드)
├── /meetings              # 내 모임 목록
│   ├── /new              # 모임 생성
│   └── /[meeting_id]     # 모임 상세
│       ├── /events       # 일정 목록
│       │   ├── /new      # 일정 등록
│       │   └── /[event_id] # 일정 상세 (평가, 정산)
│       └── /posts        # 게시글
│           └── /new      # 글쓰기 (@mention)
├── /places
│   └── /[place_id]       # 가게 상세 (모임 데이터 카드, 점수)
├── /ranking              # 동·카테고리 랭킹
├── /awards               # 어워드 (수상 이력)
├── /recommendations      # 모임 장소 추천
└── /me                   # 내 정보 (간단)
```

## 컴포넌트 매핑 (SEED 활용)

| 우리 컴포넌트 | SEED 컴포넌트 |
|---|---|
| 상단 헤더 | `TopNavigation` |
| 하단 네비 | `BottomNavigation` |
| 모임/일정/가게 카드 | `List`, `ListItem` |
| 카테고리 필터 | `SegmentedControl` |
| 동네 선택 | `SelectBox` |
| 별점 입력 | `Slider` 또는 직접 |
| 알림·토스트 | `Snackbar` |
| 로딩 | `Skeleton` |
| 모달·시트 | `BottomSheet` |
| 버튼 | `ActionButton` |
| 입력 | `TextInput`, `Field` |
| 점수 시각화 | `MannerTempBadge` 응용 |
| 어워드 배지 | `Badge` 응용 |
| 자동완성 | `TextInput` + `MenuSheet` |
| 탭 | `Tabs` |
| 라디오 | `Radio` |
